request flow

1
SDK
ward.init() patches LLM clients. Each call creates an OTel span.
2
Gateway
Validates API key via Redis. Checks rate limit. Injects tenant_id.
3
Collector
Batches spans. Applies memory limits. Exports to ClickHouse.
4
ClickHouse
Stores traces with tenant_id, tokens, cost, model info.
5
Grafana
Queries ClickHouse. Shows traces, latency, tokens, cost dashboards.

src/ward/

Python SDK core. init(), OTel tracer setup, wrapt-based instrumentation for OpenAI & Anthropic. Published as ward-sdk on PyPI.

gateway/

Go API gateway (Chi). Auth middleware, Redis key lookup (SHA-256 hashed), sliding-window rate limiter, OTLP protobuf parsing + tenant_id injection.

configs/

OTel Collector config (OTLP receivers → ClickHouse exporter), ClickHouse server config, Grafana datasource + dashboard provisioning.

infra/

Terraform HCL for AWS. VPC, ECS cluster, Fargate services (gateway, redis, collector), EC2-backed ClickHouse with EBS, ALB with TLS, auto-scaling.
